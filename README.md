# dangke-question

## 项目名称

党课题库练习网页

## 项目简介

dangke-question 是一个面向党课学习、培训复习和题库练习场景设计的网页项目。项目以本地题库数据文件为基础，提供题型筛选、关键词检索、顺序练习、随机练习、错题复习、学习记录保存、记录导出与导入等功能，帮助使用者更加高效地完成党课题库的日常练习与集中复习。

本项目采用网页形式呈现，用户无需安装复杂软件，只需通过浏览器打开网页即可使用。项目既支持电脑端浏览器，也适配手机端触屏操作，适合在电脑上进行系统复习，也适合在手机上进行碎片化刷题。

项目可直接在本地打开使用，也可以部署到 GitHub Pages 或其他静态网站服务中，实现在线访问和共享使用。

## 项目定位

本项目主要服务于以下需求：

1. 对党课题库进行集中整理和网页化展示；
2. 为学习者提供便捷的刷题、检索和复习工具；
3. 支持错题自动记录，帮助用户针对薄弱题目进行反复巩固；
4. 支持学习进度本地保存，方便用户连续练习；
5. 支持记录导出和导入，便于备份或跨设备迁移；
6. 通过静态网页部署方式，降低使用门槛，方便个人、班级或组织内部共享。

## 核心功能

### 1. 多题型练习

项目支持多种常见题型，包括：

- 单选题
- 多选题
- 判断题
- 填空题

用户可以根据复习需要选择不同题型进行专项训练，也可以在全题库范围内进行综合练习。

### 2. 题型筛选

网页左侧提供“题型筛选”功能。用户可以按照题目类型快速筛选需要练习的题目，例如只练习单选题，或只查看判断题。

该功能适合分模块复习，能够帮助使用者按照题目类型逐步掌握知识点，避免所有题目混在一起造成复习负担。

### 3. 关键词检索

项目支持通过题干或选项关键词进行快速检索。用户可以在“关键词检索”输入框中输入相关词语，系统会自动筛选包含该关键词的题目。

该功能适合用于以下场景：

- 查找某一类知识点相关题目；
- 快速定位记忆不清的题目；
- 按关键词进行专题复习；
- 在大量题目中快速找到目标内容。

### 4. 题目跳转

网页支持通过两种方式快速跳转题目：

- 输入当前结果中的序号，例如 `25`；
- 输入题目 ID，例如 `Q00025`。

该功能适合在题库数量较多时快速定位某道题，也方便老师、同学或学习小组之间根据题号进行讨论。

### 5. 顺序练习

顺序练习模式适合系统复习。用户可以按照题库原有顺序逐题完成练习，适合第一次完整学习或考前集中梳理。

在顺序练习中，用户可以稳定掌握题库整体结构，逐步完成全题库复习。

### 6. 随机练习

随机练习模式适合检测掌握程度。开启随机练习后，题目会以随机顺序出现，可以减少用户对题目顺序的依赖，更接近真实复习和自测场景。

该功能适合在已经完成一轮系统练习后使用，用于检查自己是否真正掌握题目内容。

### 7. 自动判题

不同题型采用不同的判题方式：

#### 单选题和判断题

用户点击选项后，系统会立即判断正误。

如果回答正确，页面会显示正确提示；如果回答错误，页面会显示正确答案，并自动将该题加入错题集。

#### 多选题

多选题需要用户选择多个选项。当选择数量达到答案数量后，系统会自动判题。用户也可以手动点击“提交答案”进行判题。

如果回答错误，系统会显示正确答案，并将该题自动加入错题集。

#### 填空题

填空题会直接在题干中标注答案，下方答案区也会显示“填空答案”。这种方式更适合记忆型内容的学习和巩固。

### 8. 错题集功能

项目内置错题集功能。用户在练习过程中答错的题目会自动加入错题集，便于后续集中复习。

错题功能包括：

- 答错题目自动加入错题集；
- 手动点击“错题标记”加入错题；
- 手动移出错题集；
- 点击“错题复习”后只显示错题；
- 点击“退出错题复习”返回全题库；
- 点击“清空错题”清除当前浏览器中的错题记录。

错题集功能可以帮助用户把复习重点集中在薄弱环节上，减少重复练习已掌握题目的时间。

### 9. 本地学习记录保存

网页会自动将用户的学习状态保存在当前浏览器本地。保存内容包括：

- 上次做到的题目；
- 当前筛选条件；
- 当前搜索关键词；
- 是否处于随机练习模式；
- 是否处于错题复习模式；
- 错题集内容；
- 已浏览题数。

用户下次打开同一个网页时，系统会自动恢复到上次练习位置，方便继续学习。

需要注意的是，本地记录只保存在当前浏览器中。如果更换浏览器、清理浏览器数据或使用无痕模式，原有记录可能无法保留。

### 10. 导出记录

用户可以点击“导出记录”按钮，将当前学习记录导出为本地文件：

```text
dangke-study-record.json
```

该文件可以用于备份错题集、学习进度和当前练习状态，避免因为浏览器数据清理造成记录丢失。

### 11. 导入记录

用户可以点击“导入记录”，选择之前导出的 `dangke-study-record.json` 文件，恢复对应的学习进度、错题记录和当前定位。

该功能适合以下情况：

- 更换电脑或浏览器；
- 从电脑转移到手机；
- 从手机转移到电脑；
- 备份恢复学习记录；
- 多次复习之间保留学习进度。

### 12. 电脑端与手机端适配

项目同时适配电脑端和手机端。

电脑端适合完整练习、快速跳题、关键词检索和批量错题复习。建议使用 Chrome、Edge 等现代浏览器。

手机端适合碎片化刷题，页面已经适配触屏操作。用户可以根据题干长度和选项数量选择竖屏或横屏浏览。

## 使用方法

### 本地使用

下载或克隆本项目后，进入项目中的网页目录，直接打开 `index.html` 即可使用。

如果页面打开后没有自动显示题目，请点击左侧“数据加载”，手动选择同目录下的 `questions.json` 文件。

### 在线使用

如果项目已经部署到 GitHub Pages 或其他静态网站平台，用户可以直接通过网页链接访问，无需下载项目文件。

在线部署后，手机端和电脑端都可以通过浏览器访问，更适合多人共享使用。

## 推荐使用流程

建议按照以下方式使用本项目：

1. 首先进行顺序练习，完整熟悉题库内容；
2. 按题型进行专项练习，例如先练习单选题，再练习多选题；
3. 对不熟悉的知识点使用关键词检索进行专题复习；
4. 开启随机练习，检查自己是否真正掌握题目；
5. 每次练习后进入错题复习，集中回顾答错题目；
6. 定期导出学习记录，避免浏览器数据丢失；
7. 如需跨设备使用，可通过导出和导入记录完成迁移。

## 常见问题

### 页面打开后没有题目怎么办？

请先确认 `questions.json` 与 `index.html` 是否位于同一目录下。

如果浏览器阻止本地文件自动读取，可以点击页面左侧的“数据加载”，手动选择 `questions.json` 文件。

### 为什么刷新后记录消失？

本项目的学习记录保存在当前浏览器本地。如果更换浏览器、清理浏览器缓存、清除网站数据或使用无痕模式，原有记录可能无法保留。

建议定期使用“导出记录”功能备份学习进度。

### 手机和电脑的记录为什么不同步？

当前版本没有账号系统，也没有云端同步功能。手机和电脑中的学习记录分别保存在各自浏览器中。

如果需要跨设备同步，可以先在一台设备上导出记录，再到另一台设备上导入记录。

### 可以多人共用吗？

可以。项目可以部署到 GitHub Pages 或其他静态网站平台，其他用户通过链接即可访问网页进行练习。

但需要注意，每个用户的学习记录都保存在自己的浏览器本地，不会互相同步或覆盖。

### 可以修改题库吗？

可以。题库内容存储在 `questions.json` 文件中。修改题库时，需要保持原有数据格式正确，否则网页可能无法正常读取题目。
